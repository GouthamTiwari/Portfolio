Section 01

Executive Summary

An enterprise-grade workflow analytics and monitoring system designed to streamline ticket management, team performance tracking, and operational oversight for support teams.

Purpose & Value Proposition

Sentinel is an enterprise-grade workflow analytics and monitoring system designed to streamline ticket management, team performance tracking, and operational oversight for support teams. The system provides real-time visibility into ticket queues, associate performance metrics, leave management, and on-call scheduling.

Key Value Propositions

Centralized Dashboard

Monitor team performance and ticket metrics in real-time from a single, unified interface with comprehensive analytics.

Automated Data Import

Import and analyze ticket data automatically with support for 54+ data columns from SIM ticketing system.

Real-time Metrics

Track KPIs, performance metrics, and team productivity with live data visualization and reporting.

Leave Management

Visual calendar system for tracking team availability, managing time off requests, and planning coverage.

On-Call Scheduling

Streamlined on-call scheduling with conflict detection, validation rules, and automated notifications.

Quality Assurance

Comprehensive audit tracking for QA, Lobby, and Paragon audits with detailed reporting capabilities.

Target Users

User Type Description Key Capabilities
Associates Team members who work on tickets and track performance WIP tracking, metrics viewing, NPT reports, leave requests
Team Leaders Supervisors who monitor team performance and conduct audits Performance monitoring, audit management, on-call scheduling, reporting
Administrators DevOps/Developers who manage system configuration Data import, associate management, database queries, system configuration

System Components

Sentinel consists of two main components working in harmony to provide comprehensive workflow analytics:

1. Associate Dashboard (Tampermonkey Extension)

A browser extension that integrates directly with the SIM (Simple Issue Management) ticketing system, providing associates with real-time visibility into their workload and performance.

  • Floating dashboard button integrated into SIM interface
  • Full-screen overlay with glassmorphism design
  • Real-time WIP (Work In Progress) tracking
  • Personal metrics and performance data display
  • NPT (Non-Productive Time) report generation
  • User profile with avatar and notifications

2. Admin Portal (React Web Application)

A full-featured web application with secure authentication, database integration, and comprehensive management capabilities for administrators and team leaders.

  • Dashboard Overview with ticket analytics
  • Team Performance tracking and metrics
  • Leave Tracker with calendar interface
  • On-Call Scheduler with validation
  • Audit Management (QA, Lobby, Paragon)
  • Reports generation and export
  • Data Management for imports and associates
  • Database Management with SQL interface
  • Settings and user preferences

Database Schema

PostgreSQL database with 12 tables and optimized indexes for performance:

Table Name Key Columns Purpose
users id, email, password, name, role, created_at Authentication and user management
associates id, name, pseudo_name, login_id, shift, is_leader, region, is_active, in_leave_tracker, in_on_call Team member profiles and settings
tickets 54+ columns including IssueId, Status, AssigneeIdentity, CreateDate, ResolvedDate Complete ticket data from SIM
leaves id, associate_id, date, leave_type, created_at Leave tracking records
shifts id, date, user_ids, time_start, time_end, created_at On-call scheduling data
qa_audits id, associate_id, ticket_id, score, feedback, auditor, created_at QA audit records
lobby_audits id, associate_id, observations, created_at Lobby audit records
paragon_audits id, associate_id, template_adherence, created_at Paragon audit records
import_history id, filename, total_rows, rows_inserted, rows_updated, rows_failed, imported_by, imported_at, duration_ms CSV import tracking
notifications id, title, message, type, is_read, created_at System notifications
batches id, batch_id, file_name, processed_at Batch processing records

Database Indexes

  • idx_tickets_issue_id - Fast ticket lookup by IssueId
  • idx_tickets_short_id - Quick access by ShortId
  • idx_tickets_status - Filter tickets by status
  • idx_tickets_item - Search by item
  • idx_tickets_create_date - Sort by creation date
  • idx_tickets_assigned_group - Group by assignment

Section 08

User Guides

Step-by-step instructions for using Sentinel's key features.

Getting Started

For Associates

  • Install Tampermonkey extension in Chrome/Edge browser
  • Install Sentinel Associate Dashboard script
  • Navigate to SIM ticketing system
  • Click the floating Sentinel button to open dashboard
  • View your WIP, metrics, and generate NPT reports

For Administrators

  • Navigate to Sentinel Admin Portal URL
  • Log in with authorized email and password
  • Access dashboard to view team analytics
  • Use Data Management to import tickets and manage associates
  • Configure leave tracker and on-call schedules

Common Workflows

Importing Tickets

  • Navigate to Data Management → Ticket Import tab
  • Export CSV from SIM ticketing system
  • Drag and drop CSV file into upload area
  • Review preview of first 5-10 rows
  • Click "Import" button to process
  • Monitor progress indicator
  • Review import summary (inserted/updated/failed)

Managing Leaves

  • Navigate to Leave Tracker page
  • Click "Add Leave" button
  • Select associate from dropdown
  • Choose date from calendar picker
  • Select leave type (WFH/Sick/PTO/Holiday/Comp Off)
  • Click "Save" to add leave
  • To delete: click on existing leave entry

Scheduling On-Call Shifts

  • Navigate to Schedule page
  • Click "Add Shift" button
  • Select date from calendar
  • Choose shift time (start and end)
  • Select one or two associates for the shift
  • System validates for conflicts
  • Click "Save" to create shift
  • View color-coded shifts on calendar

Conducting Audits

  • Navigate to Audits page
  • Select audit type tab (QA/Lobby/Paragon)
  • Click "Create New Audit" button
  • Fill in all required fields
  • Add scores and feedback
  • Click "Submit" to save audit
  • View audit history in table below

Term Definition
SIM Simple Issue Management - The ticketing system Sentinel integrates with

UPSERT Update or Insert - Database operation that updates existing records or inserts new ones

QA Audit Quality Assurance audit to evaluate ticket resolution quality
Lobby Audit Audit type for monitoring associate behavior and compliance
Paragon Audit Audit type for template adherence and process compliance
